PROGRAMMING PRINCIPLES

It's not easy with them, but almost impossible without them

OOP

OOP stands for Object-Oriented Programming.
For me, OOP has several advantages over procedural programming:

  1. OOP is faster and easier to execute
  2. OOP provides a clear structure for the programs
  3. OOP makes it possible to create full reusable applications with less code and shorter development time

SOLID Principles

SOLID is an acronym for the first five object-oriented design (OOD) principles by Robert C. Martin (also known as Uncle Bob).

  1. S - Single-responsiblity Principle
  2. O - Open-closed Principle
  3. L - Liskov Substitution Principle
  4. I - Interface Segregation Principle
  5. D - Dependency Inversion Principle

DRY PRINCIPLE

Basically, when you find yourself writing the same code over and over again, there may be a better way to do it...

  1. D - Don't
  2. R - Repeat
  3. Y - Yourself
